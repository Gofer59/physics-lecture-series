# Voice-Clone Workflow for QFT Lecture 1

This document describes how to synthesize per-scene audio files from `narration_transcript.txt`
using OpenVoice V2, with `voice_sample.wav` as the tone-color reference.

**Two directories are used:**

| Variable | Path | Purpose |
|---|---|---|
| `PIPELINE_DIR` | `/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline` | Holds `voice_sample.wav`, scripts, intermediate tensors, and final audio output. |
| `CONTENT_DIR`  | `/home/gofer/Documents_organise/references/video_project/qft01_v2_production` | Holds the written content: `narration_transcript.txt`, `scene_breakdown.md`, `PROMPT.md`, `MANIFEST.md`. Upload this folder to Claude Design. |

Audio lives in `PIPELINE_DIR` (not in the Claude Design upload) because Claude Design rejects
binary audio uploads.

---

## Important Caveat: Clone-of-Synthesis Limitation

`voice_sample.wav` was extracted from a prior audio file generated by Edge TTS (Emma Multilingual
Neural). It is synthetic speech, not a human recording. OpenVoice V2's tone-color converter is
designed to transfer vocal identity from a human reference recording. Cloning from synthetic
speech is technically possible but will yield lower fidelity than cloning from a real human voice:
timbre artifacts from the TTS engine are baked into the reference, and the clone will reproduce
them rather than a clean vocal identity.

**Recommended simpler path:** If you want audio that sounds exactly like Emma Multilingual, skip
OpenVoice entirely and use Edge TTS directly (see Section F below). It is lossless with respect
to Emma's voice and requires no GPU or checkpoint downloads. Use OpenVoice only if you want to
experiment with voice style modification on top of the Emma baseline, or if Emma is unavailable.

## B. Extracting the Tone-Color Embedding from voice_sample.wav

The tone-color embedding is a compact representation of the speaker's voice identity. Extract it
once and reuse it for all scenes.

```python
#!/usr/bin/env python3
# extract_tone_color.py
import torch
from openvoice import se_extractor
from openvoice.api import ToneColorConverter

PIPELINE_DIR = "/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline"
OPENVOICE_DIR = "/home/gofer/OpenVoice"
CKPT = f"{OPENVOICE_DIR}/checkpoints_v2/converter"
DEVICE = "cuda:0" if torch.cuda.is_available() else "cpu"

converter = ToneColorConverter(f"{CKPT}/config.json", device=DEVICE)
converter.load_ckpt(f"{CKPT}/checkpoint.pth")

reference_wav = f"{PIPELINE_DIR}/voice_sample.wav"
target_se, audio_name = se_extractor.get_se(reference_wav, converter, vad=True)

embedding_path = f"{PIPELINE_DIR}/voice_sample_se.pth"
torch.save(target_se, embedding_path)
print(f"Tone-color embedding saved to {embedding_path}")
```

Run:
```bash
conda activate openvoice
cd /home/gofer/OpenVoice
python /home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline/extract_tone_color.py
```

This creates `voice_sample_se.pth` in `PIPELINE_DIR`.

## C. Splitting narration_transcript.txt into Per-Scene Text Files

The narration transcript lives in `CONTENT_DIR` and uses `## Scene NN — Title` headers. Split it
into per-scene files inside `PIPELINE_DIR`:

```python
#!/usr/bin/env python3
# split_scenes.py
import re, os

PIPELINE_DIR = "/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline"
CONTENT_DIR  = "/home/gofer/Documents_organise/references/video_project/qft01_v2_production"
TRANSCRIPT = f"{CONTENT_DIR}/narration_transcript.txt"
TEXT_DIR   = f"{PIPELINE_DIR}/scene_texts"
os.makedirs(TEXT_DIR, exist_ok=True)

with open(TRANSCRIPT) as f:
    content = f.read()

# Split on scene headers
pattern = r'^## Scene (\d+)'
parts = re.split(pattern, content, flags=re.MULTILINE)
# parts = ['', '01', 'text...', '02', 'text...', ...]

for i in range(1, len(parts), 2):
    scene_num = parts[i].zfill(2)
    text = parts[i + 1].strip()
    # Remove the title line (first line after the split)
    lines = text.split('\n')
    # First line is the rest of the header (— Title); skip it
    body = '\n'.join(lines[1:]).strip()
    out_path = f"{TEXT_DIR}/scene_{scene_num}.txt"
    with open(out_path, 'w') as f:
        f.write(body)
    print(f"Wrote {out_path}")
```

Run:
```bash
python split_scenes.py
```

## D. Synthesizing Per-Scene WAV Files

For each scene, use MeloTTS (the V2 base speaker) to synthesize a draft WAV, then run the
tone-color converter to map it to the Emma voice identity.

```python
#!/usr/bin/env python3
# synthesize_scenes.py
import os, glob, torch
from melo.api import TTS
from openvoice.api import ToneColorConverter
from openvoice import se_extractor

PIPELINE_DIR = "/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline"
OPENVOICE_DIR = "/home/gofer/OpenVoice"
CKPT = f"{OPENVOICE_DIR}/checkpoints_v2/converter"
DEVICE = "cuda:0" if torch.cuda.is_available() else "cpu"
SPEED = 1.0

# Load converter and target embedding
converter = ToneColorConverter(f"{CKPT}/config.json", device=DEVICE)
converter.load_ckpt(f"{CKPT}/checkpoint.pth")
target_se = torch.load(f"{PIPELINE_DIR}/voice_sample_se.pth", map_location=DEVICE)

# MeloTTS base speaker for English (EN_NEWEST gives best quality)
tts_model = TTS(language='EN', device=DEVICE)
speaker_ids = tts_model.hps.data.spk2id
speaker_key = 'EN-US'  # adjust if your MeloTTS uses a different key
speaker_id = speaker_ids[speaker_key]
source_se = torch.load(
    f"{OPENVOICE_DIR}/checkpoints_v2/base_speakers/ses/en-us.pth",
    map_location=DEVICE
)

audio_dir = f"{PIPELINE_DIR}/audio"
os.makedirs(audio_dir, exist_ok=True)
tmp_wav = f"{audio_dir}/_tmp_melo.wav"

text_files = sorted(glob.glob(f"{PIPELINE_DIR}/scene_texts/scene_*.txt"))
for txt_path in text_files:
    scene_id = os.path.basename(txt_path).replace('.txt', '')  # e.g. scene_01
    out_wav = f"{audio_dir}/{scene_id}.wav"
    with open(txt_path) as f:
        text = f.read().strip()

    # Step 1: synthesize with MeloTTS base speaker
    tts_model.tts_to_file(text, speaker_id, tmp_wav, speed=SPEED)

    # Step 2: apply tone-color conversion
    converter.convert(
        audio_src_path=tmp_wav,
        src_se=source_se,
        tgt_se=target_se,
        output_path=out_wav,
        message="@QFT_v2"
    )
    print(f"Synthesized {out_wav}")

os.remove(tmp_wav)
print("Done.")
```

Run:
```bash
conda activate openvoice
python synthesize_scenes.py
```

Output files: `PIPELINE_DIR/audio/scene_01.wav` through `PIPELINE_DIR/audio/scene_20.wav`.

## E. Bash Wrapper (full pipeline, one command)

Save as `run_voice_synthesis.sh` in `PIPELINE_DIR`:

```bash
#!/usr/bin/env bash
set -euo pipefail

PIPELINE_DIR="/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline"
OPENVOICE_DIR="/home/gofer/OpenVoice"

cd "$OPENVOICE_DIR"

python "$PIPELINE_DIR/extract_tone_color.py"

echo "=== Step 2: Split transcript into per-scene text files ==="
python "$PIPELINE_DIR/split_scenes.py"

echo "=== Step 3: Synthesize per-scene WAV files ==="
python "$PIPELINE_DIR/synthesize_scenes.py"

```

Make executable and run:
```bash
chmod +x run_voice_synthesis.sh
./run_voice_synthesis.sh
```

## F. Alternative: Edge TTS Direct Synthesis (Recommended)

If you want the cleanest Emma voice with zero quality loss:

```bash
#!/usr/bin/env bash
set -euo pipefail

PIPELINE_DIR="/home/gofer/Documents_organise/references/video_project/qft01_v2_voice_pipeline"
CONTENT_DIR="/home/gofer/Documents_organise/references/video_project/qft01_v2_production"
TEXT_DIR="$PIPELINE_DIR/scene_texts"
AUDIO_DIR="$PIPELINE_DIR/audio"

pip install edge-tts --quiet
mkdir -p "$AUDIO_DIR"

# First split the transcript (run split_scenes.py above, or equivalent)
python "$PIPELINE_DIR/split_scenes.py"

for txt in "$TEXT_DIR"/scene_*.txt; do
    scene=$(basename "$txt" .txt)
    echo "Synthesizing $scene ..."
    edge-tts \
        --voice en-US-EmmaMultilingualNeural \
        --file "$txt" \
        --write-media "$AUDIO_DIR/${scene}.wav"
done

echo "Done. Files in $AUDIO_DIR/"
```

This produces audio perceptually indistinguishable from the original Emma voice used in the
existing QFT lecture series, and requires no GPU, no checkpoint download, and no conda
environment beyond a single pip install.
